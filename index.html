Remote Macro

For this project, I combined hardware reverse engineering with embedded software development. I began by using a logic analyzer to capture IR signals from a remote control, then wrote a C program to translate the raw ones and zeros into readable signal patterns.
After decoding the hardware signals, I moved to emulation on an Arduino. The first version could send a single signal, but through iterative design I expanded it to handle sequences of signals (macros). One design challenge I encountered was: “What if my macro opens an app that takes time to load — how do I account for that delay?”
The solution was a 2D array structure, where one index represented the signal and the other represented the delay. This made it possible to create macros with precise timing. In the next iteration, I added the ability to dynamically create, edit, and save macros using the Arduino’s EEPROM, ensuring persistence across reboots.
To make it user-friendly, I integrated a simple LCD interface, which displayed the current macro, the signal index being modified, and prompted for delay input. This allowed for on-device macro editing without needing to reflash the code.
